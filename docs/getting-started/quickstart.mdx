description: Get your first workflow running with Omium in 5 minutes

<Note>
This guide assumes you have Python 3.9+ installed. If you don't have an Omium account yet, [sign up here](https://omium.ai).
</Note>

## Prerequisites

- Python 3.9+
- An Omium account
- An API key from [app.omium.ai/api-keys](https://app.omium.ai/api-keys)

---

## Step 1: Install the SDK

```bash
pip install omium
```

<Tip>
Verify installation with `omium --version`
</Tip>

## Step 2: Initialize Omium

<Tabs>
  <Tab title="CLI (Recommended)">
    ```bash
    omium init
    ```
    
    You'll be prompted to enter your API key. Config is saved to `~/.omium/config.json`.
  </Tab>
  <Tab title="Python">
    ```python
    import omium
    
    omium.init(api_key="om_xxx")
    ```
  </Tab>
</Tabs>

## Step 3: Instrument Your Framework

<Tabs>
  <Tab title="LangGraph">
    ```python
    import omium
    
    omium.init(api_key="om_xxx")
    omium.instrument_langgraph()
    
    # Your LangGraph code works unchanged
    from langgraph.graph import StateGraph
    
    graph = StateGraph(...)
    app = graph.compile()
    result = app.invoke({"input": "Hello"})
    ```
  </Tab>
  <Tab title="CrewAI">
    ```python
    import omium
    
    omium.init(api_key="om_xxx")
    omium.instrument_crewai()
    
    # Your CrewAI code works unchanged
    from crewai import Crew, Agent, Task
    
    crew = Crew(agents=[...], tasks=[...])
    result = crew.kickoff()
    ```
  </Tab>
  <Tab title="Manual Tracing">
    ```python
    import omium
    
    omium.init(api_key="om_xxx")
    
    @omium.trace("process_data")
    def process_data(data):
        return transformed_data
    
    @omium.checkpoint("important_step")
    def important_step():
        return result
    ```
  </Tab>
</Tabs>

## Step 4: View in Dashboard

<Info>
After running your workflow, all traces and checkpoints appear automatically in the dashboard.
</Info>

Visit [app.omium.ai](https://app.omium.ai) to see:

- **Executions** - All workflow runs with status
- **Traces** - Step-by-step execution details
- **Checkpoints** - Recovery points for failed runs
- **Replay** - Re-run failed executions from checkpoints
