The Omium CLI (`omium`) provides commands for configuration, running workflows, and managing executions.

## Quick Reference

| Command | Description |
|---------|-------------|
| `omium init` | Initialize SDK configuration |
| `omium run <script.py>` | Run script with tracing |
| `omium list` | List recent executions |
| `omium show <id>` | Show execution details |
| `omium logs <id>` | Stream execution logs |
| `omium replay <id>` | Replay from checkpoint |
| `omium tui` | Launch Terminal UI |

---

## Installation

<Tip>
The CLI is automatically installed with the SDK.
</Tip>

```bash
pip install omium
omium --version  # Verify installation
```

## Configuration Commands

### `omium init`

Initialize Omium SDK configuration. This is the first command to run.

```bash
omium init
```

**Options:**
- `--api-key TEXT` - API key (recommended for Windows/PowerShell)
- `--api-url TEXT` - API URL (default: https://api.omium.ai)
- `--skip-verify` - Skip API key verification

**Example:**
```bash
# Interactive mode
omium init

# Non-interactive (Windows recommended)
omium init --api-key omium_xxxxx --api-url https://api.omium.ai
```

### `omium configure`

Update existing configuration.

```bash
omium configure [OPTIONS]
```

**Options:**
- `--api-key TEXT` - New API key
- `--api-url TEXT` - API URL
- `--region TEXT` - Region (us-east-1, eu-west-1)
- `--interactive / --no-interactive` - Interactive prompts

## Workflow Commands

### `omium run`

Run a Python script or workflow with Omium instrumentation.

```bash
omium run <script.py|workflow.json> [OPTIONS]
```

**Options:**
- `--project TEXT` - Project name
- `--env KEY=VALUE` - Environment variables (can repeat)
- `--no-trace` - Disable automatic tracing
- `--execution-id TEXT` - Custom execution ID

**Examples:**
```bash
# Run Python script with auto-instrumentation
omium run my_agent.py

# Run with environment variables
omium run my_agent.py --env OPENAI_API_KEY=sk-xxx --env DEBUG=true

# Run workflow JSON (legacy)
omium run workflow.json
```

### `omium init-workflow`

Create a workflow template file.

```bash
omium init-workflow --type <TYPE> --name <NAME>
```

**Options:**
- `--type TEXT` - Workflow type: `crewai`, `langgraph`, `basic`
- `--name TEXT` - Workflow name
- `--output TEXT` - Output file path

**Example:**
```bash
omium init-workflow --type langgraph --name my-agent
```

## Execution Commands

### `omium list`

List recent executions.

```bash
omium list [OPTIONS]
```

**Options:**
- `--status TEXT` - Filter by status (running, completed, failed)
- `--workflow-id TEXT` - Filter by workflow
- `--limit INT` - Number of results (default: 20)

**Example:**
```bash
omium list
omium list --status failed --limit 10
```

### `omium show`

Show detailed information about an execution.

```bash
omium show <EXECUTION_ID>
```

**Example:**
```bash
omium show exec-abc123
```

### `omium logs`

Stream logs from a running execution.

```bash
omium logs <EXECUTION_ID> [OPTIONS]
```

**Options:**
- `--follow / --no-follow` - Stream live logs (default: follow)
- `-n, --tail INT` - Number of recent logs to show

**Examples:**
```bash
omium logs exec-123           # Stream live
omium logs exec-123 --no-follow  # Show last logs and exit
omium logs exec-123 -n 100    # Show last 100 logs
```

### `omium replay`

Replay an execution from a checkpoint.

```bash
omium replay <EXECUTION_ID> [OPTIONS]
```

**Options:**
- `--checkpoint-id TEXT` - Specific checkpoint to replay from

**Example:**
```bash
omium replay exec-abc123
omium replay exec-abc123 --checkpoint-id cp-456
```

### `omium status <execution_id>`

Watch execution status until it completes.

### `omium traces projects` / `omium traces list` / `omium traces show <trace_id>`

List projects with traces, list traces (optionally by `--project`), or show one trace.

```bash
omium traces projects
omium traces list --project my-agent
omium traces show <trace-id>
```

### `omium failures list` / `omium failures show <execution_id>`

List failed runs or show failure details for an execution.

### `omium checkpoints list <execution_id>`

List checkpoints for an execution (uses local checkpoint manager by default).

### `omium rollback <execution_id> <checkpoint_id>`

Rollback an execution to a checkpoint.

## Billing and analytics

Show runs, success rate, and related metrics.

### `omium scores create`

### `omium scores list [--trace-id] [--name] [--limit]`

## Export and templates

### `omium export-crew <source>` / `omium export-langgraph <source>`

Export a CrewAI or LangGraph workflow from a Python file or module.

### `omium new [template] [name]`

Create a new workflow from a template.

## Project Commands

### `omium project init`

Create an `omium.toml` and workflows directory in the current (or specified) directory.

```bash
omium project init [OPTIONS]
```

**Options:**
- `--name TEXT` - Project name
- `--dir TEXT` - Project directory (default: current)

### `omium project push`

Push local project and workflows to Omium cloud. Makes the project visible on the [Automations dashboard](https://app.omium.ai/automations).

```bash
omium project push [OPTIONS]
```

**Options:**
- `--create` - Create project on cloud if it does not exist

**Example:**
```bash
omium project init --name my-agent
omium project push
```

### `omium project info` / `omium project workflows`

Show project info or list workflows from `omium.toml`.

## TUI Mode

### `omium tui`

Launch the interactive Terminal UI for browsing executions.

### `omium chat`

Start an interactive chat session (e.g. for building workflows).

### `omium completions <shell>`

Print shell completion script. Shells: `bash`, `zsh`, `fish`, `powershell`.

```bash
omium completions powershell
```

## Environment Variables

| Variable | Description |
|----------|-------------|
| `OMIUM_API_KEY` | API key (overrides config file) |
| `OMIUM_API_URL` | API URL |
| `OMIUM_DEBUG` | Enable debug logging |

## Configuration File

Config is stored at `~/.omium/config.json`:

```json
{
  "api_key": "omium_xxx",
  "api_url": "https://api.omium.ai",
  "region": "us-east-1"
}
```

<Tip>
Do not add `/api/v1` to `api_url`; the CLI appends it when needed.
</Tip>
